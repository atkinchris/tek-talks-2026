Why one byte breaks it
kernel.img → a3f9c2d1… ok
kernel.img (modified) → ff02b8e4… sig invalid

The signature covers the SHA-256 hash of the full image. Any change produces a completely different hash.

Consequence

Verification fails → device halts.

You can't patch the kernel on disk, swap the rootfs, or inject code — the chain will catch it.
