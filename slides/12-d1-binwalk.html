Binwalk

binwalk tells us what it can read - and what it can't

Scan the binary blob and it identifies known signatures. The rootfs is a problem.

$ binwalk firmware.bin

DECIMAL HEXADECIMAL DESCRIPTION
-------- ----------- --------------------------------
0 0x0 uImage header, OS: Linux, ...
64 0x40 Linux kernel ARM boot image
2097152 0x200000 High entropy data (encrypted?)

The kernel is readable. The rootfs - where all the interesting application code lives - is not.
