Firmware

A firmware image is built from independent parts

Each comes from a different source - and has a different attack surface.

Stage 1
Bootloader

Runs first. Initialises hardware and loads the kernel into memory. Usually U-Boot or a vendor-customised fork.

Vendor-written
Stage 2
Kernel

The Linux kernel, often an old LTS release with vendor patches applied. Manages memory, processes, and device drivers.

Linux + vendor patches
Stage 3
Root filesystem

Everything else - the applications, web UI, config files, and the games themselves. Often a SquashFS image. By far the largest part.
